# moyeoradeal-admin 적용 가이드 — 고도화된 업무관리 OS 이식

이 폴더(`moyeora-os-portable/`)는 **넷폼알앤디에서 고도화된 POUR OS(업무관리 앱)의 이식용 포터블 버전**입니다.
앱 로직·기능은 그대로, **브랜드 값만 파일 하나(`src/brand.config.js`)로 분리**해 뒀습니다.
`moyeoradeal-admin` 저장소의 `moyeora-os/` 를 이 폴더 내용으로 교체하면, 지금 쓰던 데이터를 유지한 채 최신 기능으로 올라갑니다.

> ⚠️ 저는 다른 GitHub 조직(`moyeoradeal-lang`)에 직접 push 권한이 없어, 여기(`pour-construction-form`)에 포터블 버전으로 만들어 둡니다.
> 아래 3단계대로 `moyeoradeal-admin` 쪽에서 반영해 주세요. (또는 moyeoradeal-admin을 소스로 새 세션을 열면 제가 거기서 직접 반영합니다.)

---

## 딱 3단계

### 1) 파일 교체
`moyeoradeal-admin/moyeora-os/` 안의 아래 파일을 이 폴더 것으로 덮어씁니다.
```
src/            (App.jsx, firebase.js, durable.js, crmOperatorSync.js, kpi.js, launch.js, main.jsx …)
functions/api/  (coach.js, backup.js)
index.html · package.json · vite.config.js
src/brand.config.js   ← 새로 추가되는 핵심 파일
```
> 새로 추가된 건 **`src/brand.config.js` 하나**뿐이고, 나머지는 기능 고도화분 반영입니다.

### 2) `src/brand.config.js` 채우기 (여기만 바꾸면 됨)
이 파일 하나가 브랜드의 전부입니다. 아래 표대로 채우세요.

| 필드 | 무엇 | moyeoradeal 값 |
|------|------|----------------|
| `firebase` | **moyeoradeal 전용** Firebase 콘솔의 웹 SDK 값 | 콘솔 > 프로젝트 설정 > 웹 앱에서 복사 |
| `dataNamespace` | Firestore 상태 문서가 저장되는 **컬렉션명** | ⚠️ **아래 3-A 필독** |
| `storagePrefix` | 기기 로컬 저장 키 접두사 | 예: `moyeora-os` (기기 임시 미러라 아무 값이나 무방) |
| `appName` / `appSubtitle` / `orgLabel` / `logoLetter` | 헤더·로고 표시 이름 | 예: `모여라딜 OS` / `업무관리` / `모여라딜` / `M` |
| `backupTag` | JSON 백업 파일 태그·파일명 | 예: `moyeora-os` |
| `accent` / `accentLight` / `accentDark` / `navy` | 브랜드 포인트 색 | 브랜드 컬러 코드 |
| `coachModel` | AI 코치가 쓰는 Claude 모델 | 기본값 유지 권장 |
| `crmEmbed.enabled` | POUR스토어 CRM iframe 연동 여부 | moyeoradeal은 보통 `false` |

### 3) 빌드 & 배포 (기존과 동일)
```bash
npm install
npm run build          # → dist/ (정적)
# Cloudflare Pages: Build=npm run build, Output=dist, 환경변수 ANTHROPIC_API_KEY
```
GitHub 연결형 Pages면 push만 하면 자동 빌드됩니다.

> (선택) **GitHub 자동 백업**(`functions/api/backup.js`)을 쓰려면 Cloudflare Pages 환경변수에
> `GITHUB_TOKEN`, `GITHUB_BACKUP_REPO`(예: `moyeoradeal-lang/moyeoradeal-admin`),
> `GITHUB_BACKUP_DIR`(예: `moyeora-os-backups`)를 moyeoradeal 값으로 넣으세요. 안 넣으면 그냥 비활성입니다.

## ⚠️ 3-A. 데이터 연속성 — `dataNamespace` (가장 중요)

앱은 팀 데이터를 Firestore에 `{dataNamespace}/state-{키}` 문서들로 저장합니다.
**지금 moyeora-os가 쓰던 컬렉션명과 똑같이 맞춰야 기존 데이터가 그대로 보입니다.**

- 확인법: 현재 `moyeora-os/src/firebase.js` 에서 `doc(db, "___", "state")` 의 `"___"` 값.
- 원본 POUR OS를 그대로 fork 했다면 대개 **`"pour-os"`** 그대로일 확률이 높습니다.
  → 그래서 기본값을 **`"pour-os"`** 로 뒀습니다. 확실하지 않으면 **그대로 두고** 배포 후 데이터가 보이는지 확인하세요.
- 값을 바꾸면 이전 데이터가 **안 보이게** 됩니다(삭제되는 건 아님, 다른 컬렉션을 볼 뿐). 되돌리면 다시 보입니다.

`firebase`(프로젝트)와 `dataNamespace`(컬렉션)는 **반드시 moyeoradeal 것**으로. POUR 값으로 두면 두 브랜드가 같은 데이터를 공유하게 됩니다.

## Firestore / Storage 보안 규칙

앱은 아래 경로를 읽고 씁니다. moyeoradeal Firebase 규칙에 없다면 추가하세요.
```
// Firestore — {dataNamespace} = brand.config 의 dataNamespace 값
match /{namespace}/{doc} { allow read, write: if true; }   // 예: match /pour-os/{doc}
// Storage — task 사진 첨부
match /task-attachments/{allPaths=**} { allow read, write: if true; }
```
> 현재 앱은 자체 방식(서버측 Auth 없음)으로 동작합니다. 보안 강화가 필요하면 별도 협의.

## 이 버전의 고도화 요약 (moyeoradeal이 받게 되는 것)

- **3중 데이터 안전망**: Firestore 분할 문서 + localStorage 미러 + IndexedDB 영구 미러·시점 스냅샷
- **소프트삭제 휴지통**(`trash`) — 영구삭제 없이 복구 가능, 빈 원격으로 로컬 덮어쓰기 방지(백지 가드)
- 컬렉션별 **1MiB 한도 가드**, 활동기록 자동 정리
- **KPI ▸ 데이터** 화면: 휴지통·전체 백업(JSON)·로컬(IndexedDB) 복구 UI
- **자동화 엔진**(`launch.js`): 업무 완료 시 후속 업무 자동 생성·연쇄 완료(선행연결)
- **프로세스 템플릿 → SKU 출시 프로젝트 자동 생성**, 구간(세그먼트) 집계 → KPI 통합
- **AI 코치**를 `/api/coach` 프록시로 이식(브라우저에 API 키 노출 안 함)
- 매출/KPI 자동집계 흐름(선행지표=진척% / 결과=매출 분리), 주간 마감 입력 UX
