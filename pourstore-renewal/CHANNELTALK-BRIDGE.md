# 채널톡 ↔ POUR Claude 봇 브리지 설계 (3차)

> **목표**: Cafe24 자사몰의 고객 채팅은 **채널톡(이미 운영 중)** 으로 받고,
> 1차 자동응답은 **우리 Claude 봇(챗봇센터의 b2c/b2b/store 프로필 + 지식베이스)** 이 처리.
> 못 풀면 채널톡 **상담원(사람)** 으로 전환.
>
> 우리 자산(마스터프롬프트·KB·교정 루프·비용 대시보드)을 그대로 두뇌로 재사용한다.

---

## 1. 아키텍처

```
고객 (Cafe24, 채널톡 위젯)
   │ 메시지 작성
   ▼
채널톡 (UI·상담원 인박스·핸드오프 담당)
   │ Webhook (userChat 메시지 이벤트)
   ▼
Cloudflare 워커  ┌─ 시크릿 검증(채널톡 서명)
 (신규: channeltalk-bridge.js) ├─ 어떤 봇 프로필 쓸지 결정 (b2c/b2b/store)
   │                          ├─ chatbot-kb 조회 → 시스템프롬프트 조립 (기존 buildSystemPrompt 재사용)
   │                          ├─ Claude 호출 (기존 claude-proxy 로직 재사용)
   │                          └─ 핸드오프 판단
   ▼
채널톡 Open API
   ├─ 봇 메시지로 답변 회신
   └─ 필요 시 "상담원 연결"(팀/매니저 배정)
   ▼
Firestore (chatbot-sessions 에 로그 저장 → 대시보드 집계)
```

> 핵심: 채널톡은 **프론트(접점·상담원·운영)**, 우리 워커는 **두뇌(답변·KB·로깅)**.

---

## 2. 사전 준비물 (컴퓨터에서, 채널톡 관리자)

1. **채널톡 Open API 키**: 채널톡 관리자 → 설정 → 개발자/API → Access Key / Secret 발급
2. **Webhook 등록**: userChat 메시지 수신 이벤트 → 우리 워커 URL 등록
3. **봇/매니저 식별자**: 봇이 보낼 발신 주체(매니저 또는 봇 프로필)
4. (선택) **운영시간/팀**: 핸드오프 시 배정할 상담팀

> 채널톡 API의 정확한 엔드포인트·이벤트명은 발급 화면/문서에서 확인 후 워커에 반영
> (Open API + Webhook 기반. 메시지 수신 → 우리가 답 생성 → userChat에 메시지 작성).

---

## 3. 만들 것 (우리 쪽)

### A. `workers/channeltalk-bridge.js` (신규 Cloudflare 워커)
- `POST /webhook` : 채널톡 webhook 수신
  - 서명/시크릿 검증
  - 봇 프로필 선택 (채널 또는 메시지 맥락 → b2c/b2b/store)
  - Firestore에서 봇 프로필 + 관련 KB 조회 (또는 캐시)
  - `buildSystemPrompt` + KB 주입 → Claude 호출 (claude-proxy 로직 공유)
  - 답변을 채널톡 Open API로 회신
  - 세션/메시지를 `chatbot-sessions`에 저장 (channel:'channeltalk')
- 핸드오프 트리거: 아래 4번 규칙

### B. 챗봇센터(`chatbot.html`) 보강
- 봇 프로필에 `channel: 'channeltalk'` 옵션/매핑
- 대화 로그·대시보드에 채널톡 세션도 같이 집계 (channel 필드만 추가 — 기존 구조 그대로)
- (선택) "핸드오프율" 지표 추가

### C. Cafe24 퀵배너
- 퀵배너에 **"💬 실시간 상담"** 버튼 → 채널톡 메신저 오픈
  ```js
  // 채널톡 부트 스크립트 로드 후
  ChannelIO('showMessenger');
  ```
- 진단·추천 같은 가이드 플로우는 채널톡 대화 안에서 우리 봇이 질문하며 진행

---

## 4. 사람(상담원) 전환 규칙 — 중요

봇이 다음이면 **즉시 상담원 연결**:
- 고객이 "상담원/사람/직원 연결" 요청
- 봇 답변 신뢰도 낮음(지식 매칭 0 + 추측 금지 → "확인 후 안내")
- 견적/계약/클레임 등 **금액·법적** 사안 (가드레일상 봇이 단정 금지)
- 운영시간 내 → 상담원 배정 / 운영시간 외 → "남겨주시면 영업시간에 연락" 안내

> 고객봇 가드레일(견적금액·타사·근거없는 보증·개인정보 직접요청 금지)은
> 챗봇센터의 b2c/b2b/store 프로필에 이미 시드돼 있음 → 그대로 적용.

---

## 5. 비용

- 채널톡 운영비: 이미 지출 중(변동 없음)
- 추가 = Claude 토큰비뿐. 고객 메시지 1건당 약 15~40원(Haiku)
- 대시보드의 **고객용 비용 카드**에 채널톡 트래픽이 합산되어 실시간 추적
- 절감책: 단순/반복 질문은 KB 즉답(짧은 출력), 복잡한 것만 긴 답변 / 필요 시 Sonnet

---

## 6. 단계

- **지금(1차)**: 채널톡은 사람 상담 + 기본 자동응답 그대로. 우리 봇은 내부 팀봇만.
- **3차-A**: 브리지 워커 MVP — 채널톡 메시지 → Claude 1차 답변 → 회신 (핸드오프는 키워드만)
- **3차-B**: 핸드오프 규칙 고도화 + 대시보드 채널톡 지표 + 진단·추천 플로우

---

## 7. 보안·주의

- 채널톡 Access Secret·Anthropic 키는 **워커 시크릿**에만 보관(클라이언트 노출 금지)
- Webhook은 채널톡 서명 검증으로 위조 차단
- 고객 PII(이름·연락처)는 채널톡이 1차 관리 → 우리 로그에는 마스킹 저장 권장
- Firebase 보안규칙 변경 필요 시 사전 협의 (CLAUDE.md 10-3 E)
