# 상품(SKU) 카탈로그 설계 — POUR스토어 챗봇

> 참고 사진(POUR스토어_탑코트재 파생 매트릭스) 기반.
> 목표: 상품을 SKU 단위로 잘게 등록하고, 추천 플로우(니즈→솔루션→상품)가 이 SKU를 가리키게 한다.

---

## 1. 핵심 원칙 — 분리하되 연결

```
(A) 추천 플로우 [chatbot-reco]        (B) 상품 SKU 카탈로그 [신규]
   건물유형 → 니즈 → 솔루션              부위×외장재×문제 × 패키지단계 → SKU
                     └ 상품추천 ──참조──→ SKU 코드
                                                 │
                                          (C) 기존 products [products]
                                          채널 상품번호·재고 (admin 상품관리)
```

- 추천 플로우의 "상품추천"은 **SKU 코드만 참조** (상품명·링크 중복입력 X)
- SKU 카탈로그는 **기존 `products` 컬렉션과 매핑**(채널 상품번호·가격·재고 일원화)

---

## 2. SKU 분류 체계 (참고 사진)

### 2-1. 분류 축
- **외장재/부위 (컬럼)**: 슬라브 · 슁글 · 기와 · 칼라강판/징크 · 균열 · 재도장 · 배수로/베란다 · 지하주차장 · 이음부 …
- **패키지 단계 (행)**: 
  - **풀패키지** (아파트 라인 / 일반 저층 라인)
  - **부분 패키지** (예: 코트재+크랙시트)
  - **단순 코팅 / 단품** (탑코트재안, 코트재안 등)
- **제품군**: 탑코트재 파생 라인 (콘크리트강화재, 이음부세트, 풀패키지 …)

### 2-2. SKU 데이터 모델 (`chatbot-skus` 신규)
```javascript
{
  id,
  code,            // 내부 SKU 코드 (예: TC-SLAB-FULL-APT)
  name,            // 표시명 (예: 아파트 옥상 풀패키지)
  line,            // 제품 라인 (예: 탑코트재 파생)
  surfaces: [],    // 외장재/부위 태그 (슬라브, 슁글, 콘크리트 …)
  problems: [],    // 문제 태그 (누수, 균열, 탈락 …)
  tier,            // 패키지 단계: 'full' | 'partial' | 'single'
  building: [],    // 적용 건물 (아파트/저층 등, 선택)
  components: [],  // 구성품 (코트재, 크랙시트, 이음부세트 …)
  image,           // 대표 이미지 URL
  price,           // 가격(선택, 또는 products 참조)
  productRef,      // 기존 products 컬렉션 doc id / 채널 상품번호
  link,            // 자사몰 구매 링크
  active: true,
  createdAt, updatedAt
}
```

---

## 3. 화면 (2차)

### 3-1. SKU 카탈로그 — 매트릭스 뷰 (사진처럼)
- 컬럼 = 외장재/부위, 행 = 패키지 단계
- 셀 = 해당 SKU 카드(이미지·이름·구성품)
- 셀 클릭 → SKU 편집(태그·구성품·이미지·products 연결)
- 필터: 라인 / 건물 / 문제

### 3-2. 추천 플로우 연결
- 니즈 편집의 "추천 상품"에서 **SKU 검색·선택**(직접 타이핑 대신)
- 선택한 SKU의 surfaces/problems가 니즈 키워드와 자동 매칭 추천

---

## 4. 봇 동작 (하이브리드)
- 고객 니즈 매칭 → 솔루션(친근 설명) → 연결된 SKU 추천
- 추천 시 **패키지 단계별 옵션** 제시 가능 (풀패키지 vs 부분 vs 단품)
  → 예산/범위에 따라 고객이 고르게
- 공법·시방서는 요청 시에만 전달 (기존 규칙 유지)

---

## 5. 단계
- **지금**: 추천 플로우 = 건물유형별 보기 (니즈→솔루션→상품) ✅
- **2차-A**: `chatbot-skus` 매트릭스 카탈로그 등록 화면
- **2차-B**: 니즈 ↔ SKU 참조 연결 + 기존 products 매핑
- **2차-C**: 봇이 패키지 단계 옵션으로 추천

---

## 6. 확인 필요 (사용자 결정)
1. SKU를 **신규 `chatbot-skus`로 따로** 둘지 vs **기존 `products` 확장**으로 갈지
2. 패키지 단계 구분(풀/부분/단품)을 SKU 속성으로 둘지, 묶음상품을 별도 SKU로 만들지
3. 매트릭스 컬럼(부위/외장재) 최종 목록 확정
