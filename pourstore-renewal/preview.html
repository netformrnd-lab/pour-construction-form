페이지 관리

페이지를 선택해 섹션을 등록·정렬하고, 변경 사유와 함께 버전 이력을 남기세요.

사용 방법
  1. 페이지 선택 — 왼쪽 사이드바에서 작업할 페이지를 클릭
  2. 섹션 추가/편집 — + 섹션 추가 또는 카드의 편집으로 HTML 입력
  3. 변경 사유 작성 — 편집 저장 시 변경 사유 필수 (이력에 기록)
  4. 순서 변경 — 카드 좌측 ⋮⋮ 그립을 잡고 드래그
  5. 컨펌 워크플로우 — 카드 좌측 상태 칩 클릭 → ⚙ 작업중·✋ 컨펌 요청(실행자) → ✅ 승인 완료 또는 ↻ 재수정 요청(관리자). 재수정 피드백은 피그마에 작성. 모든 전환은 이력에 자동 기록
  6. 이력 확인/복원 — 이력 버튼에서 이전 버전 미리보기·복원, 변경 사유 수정
  7. 전체 시안 보기 — 전체 시안 바로보기로 모든 섹션을 합쳐 미리보기. 왼쪽에서 섹션별 이전 시안을 모달로 열고, 오른쪽에서 섹션별 댓글을 바로 확인·작성할 수 있습니다 (필요 시 ↗ 새 창으로 열기)
  8. 이력 보관 — 모든 버전 영구 보관, 자동 삭제 없음. 1년 경과 항목은 상단 알림에서 직접 선택해 정리 (직전 버전은 항상 보호)
🔧 추후 적용 가능 기능 필요 시 요청 → 활성화
  • 카페24용 인라인 CSS 자동 변환 — Tailwind/외부 폰트 등 외부 의존성을 인라인 스타일로 변환해 카페24에 바로 붙여넣어도 동일하게 보이도록
  • 이미지 직접 업로드 — 섹션 편집 중 이미지를 Firebase Storage에 업로드하고 URL 자동 삽입
  • 버전 비교(diff) — 두 버전 사이 변경된 줄/속성을 색상으로 시각화
  • 사용자·권한 분리 일부 적용 — 담당자 등록·내 이름 설정 완료. 남은 작업: 페이지·섹션 단위 잠금, 작업 내역 사용자별 분리.
  • 코멘트/리뷰 워크플로우 일부 적용 — 섹션·페이지 단위 피드백 + 작업여정 타임라인 완료. 남은 작업: 다중 컨펌 단계, @멘션 알림.
  • 실 사이트 자동 반영 — 컨펌된 섹션만 묶어 카페24 API/Webhook으로 자동 게시
  • 휴지통(soft delete) 일부 적용 — 피드백 7일 휴지통 + 자동 영구 삭제 완료. 남은 작업: 섹션·페이지 삭제 시에도 휴지통 유예.
  • 변경 알림 — 동기화 시 SMS/이메일로 변경 내용 요약 발송
데이터 저장 안내
  • Firebase에 자동 저장됩니다 (프로젝트: pour-app-new · 컬렉션: pourstore-renewal-builder).
  • 모든 변경(편집·드래그·컨펌·이력 사유)은 저장 직후 서버에 업로드되고, 다른 PC·브라우저에서 열면 자동으로 같은 화면이 나타납니다.
  • 오프라인 상태에서도 작업할 수 있고, 다시 연결되면 변경 내용이 자동으로 동기화됩니다 (브라우저에 임시 저장).
  • 상단 우측 표시등으로 동기화 상태를 확인하세요 — 동기화됨 · 저장 중 · 오프라인/오류.

ⓘ 카페24 상품번호는 이미 등록된 상품에 매핑하기 위한 식별자입니다. 자동 생성되지 않습니다.
마스터의 사실 카드 중에서 이 상품에서 강조할 항목을 선택하세요. 이 선택은 ④ 산출물 단계에서 슬롯 카피 생성 시 우선순위로 사용됩니다.
선택 안 한 사실도 마스터에서 그대로 상속됩니다 — 강조는 "이 상품에서 특별히 부각할 것"의 의미.
상품이 풍기는 분위기 프리셋을 선택하면 메인 컬러·이미지 스타일·키워드가 자동 세팅됩니다. 같은 POUR코트재라도 목재페인트(따뜻)와 철재페인트(차가움)는 다른 vibe.
마스터 사실 + 강조 사실 + vibe → 상세페이지 슬롯 카피를 자동 생성합니다. 인스턴스(상세페이지)는 별도 컬렉션에 저장되고 이 상품에 연결됩니다.

모제품의 적용 가능 기재(substrates)를 기반으로 자식 상품을 자동 제안합니다. 체크한 항목만 상품(Listing)으로 일괄 생성됩니다.

제안 방식 선택
제안된 자식 상품 — 체크한 것만 생성
마스터 선택 시 적용기재가 표시됩니다.

🎓 처음 사용하시나요? — 5분 가이드

POUR스토어는 모제품 1개 → 자식 상품 N개 → 상세페이지 흐름으로 작업합니다.
아래 4단계만 따라 하면 첫 상품 상세페이지가 나옵니다.

1
초기 세팅 (첫 1회만)
대시보드 🚀 초기 세팅 섹션의 4개 항목을 차례로 완료하세요.
· Claude API 키 — sk-ant-... 입력 (Anthropic 콘솔에서 발급)
· POUR 기본 템플릿 — 14섹션 뼈대 자동 추가
· 첫 마스터 제품 — 제품명 + 카탈로그 PDF 업로드
· 사실 카드 추출 — 🤖 버튼 누르면 Claude가 자동 추출
2
상품(Listing) 만들기 — 분기 제안 추천
B 단계 → 🪄 분기 제안 클릭
· 빠른 제안 — 키워드 사전 기반 (즉시·무료)
· AI 제안 — Claude가 사실 종합 분석 (정교, 30초)
같은 모제품에서 목재페인트·철재페인트·벽돌페인트… 자동 생성됨.
3
상세페이지 자동 생성
상품 편집기 → ④ 산출물 탭 → 템플릿 선택 → 🤖 자동 생성
마스터 사실 + 강조 + vibe로 슬롯 카피가 자동 채워집니다.
결과는 📄 상세페이지 목록에서 확인·수정 가능.
4
카페24 붙여넣기
상세페이지 편집 화면에서 전체 HTML 복사
→ 카페24 상품 상세 영역에 붙여넣기 → 끝!
상품마다 카페24 상품번호를 ① 기본정보 탭에 입력해두면 매핑 추적 가능.
⚠ 시작 전 한 번만 확인 (관리자)
  • Cloudflare Worker 배포 — npx wrangler deploy --config workers/wrangler.claude-proxy.toml 후 npx wrangler secret put WORKER_SECRET. Anthropic API 키는 페이지 설정에서 직접 입력 (솔라피식).
  • Firestore 보안 규칙 — 신규 컬렉션 pourstore-renewal-products · pourstore-renewal-listings 의 read/write 권한이 풀려 있어야 합니다 (기존 pourstore-renewal-builder와 동일 규칙 적용).
  • 네트워크 — 워커 미배포 또는 URL 오타 시 모든 Claude 호출이 실패합니다. 설정 모달의 "테스트" 버튼으로 먼저 확인하세요.
💡 자주 묻는 질문
  • "카탈로그가 100페이지 넘어요" — 해당 제품 부분만 1~2장 뽑아서 PDF로 만드세요. 사실 추출 정확도 ↑.
  • "같은 목재인데 외장·실내가 다른데요" — 🤖 AI 제안을 쓰세요. 같은 기재도 용도별로 자동 분기됩니다.
  • "이미지는 어떻게 채우나요" — 텍스트 슬롯이 채워진 뒤 마누스 일괄 생성으로 N개 인스턴스의 이미지를 한 번에.
  • "마스터 사실을 수정하면 자식 상품에도 반영되나요" — 새로 생성하는 상세페이지에는 반영. 기존 상세페이지는 다시 자동 생성하면 갱신.

⚙ 마누스 워커 설정

Cloudflare Workers에 등록한 pour-manus-proxy의 URL과 빌더용 인증 시크릿을 입력하세요. Firestore app-config/manus 에 저장됩니다.
⚠ 이 시크릿은 Firestore에 평문으로 저장됩니다 (현재 보안규칙은 인증 없이 read 허용). 외부의 무차별 워커 호출만 막는 1차 방어선입니다.
