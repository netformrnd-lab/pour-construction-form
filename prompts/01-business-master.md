## §1. 회사 구조

```
넷폼알앤디 (NetformR&D) — 건축물 유지보수 전문 기업 (모회사)
│
├── POUR솔루션 — 의뢰자 타겟
│   ├── 타겟: 아파트 관리소장, 관공서 시설담당, 일반건물 건물주/시설관리자
│   ├── 건물 유형 6종
│   │   ├── 아파트 (관리소장, 입주자대표)
│   │   ├── 관공서 (시설담당 공무원)
│   │   ├── 상가/오피스텔 (건물주, 시설관리)
│   │   ├── 공장/창고 (시설담당자)
│   │   ├── 학교/병원 (시설팀)
│   │   └── 기타 건축물
│   ├── 공법 23종
│   │   ├── 방수 6종 (우레탄방수, 시트방수, 도막방수, 침투방수, 주입방수, 지하방수)
│   │   ├── 도장 5종 (외벽도장, 내벽도장, 바닥도장, 방청도장, 특수도장)
│   │   ├── 보수보강 3종 (균열보수, 단면보수, 구조보강)
│   │   ├── 토목 4종 (교량보수, 터널방수, 옹벽보수, 지하구조물)
│   │   └── 스토어 5종 (셀프방수킷, 균열보수제, 방수테이프, 코팅제, 보조자재)
│   └── 영업 채널
│       ├── 박람회 (BEXCO 등 건축박람회 — 태블릿 상담앱 활용)
│       └── 상시 아웃바운드 (전화/방문/메일 영업)
│
├── POUR공법 — 시공사 타겟
│   ├── 타겟: 종합건설사, 전문건설사, 방수업체
│   ├── 영업 방식
│   │   ├── 입찰 (나라장터/조달청)
│   │   ├── 수의계약
│   │   ├── 하도급
│   │   └── 기술제안 (VE 제안)
│   └── 핵심 가치: 특허 공법, 기술인증, 시방서 보유
│
├── POUR스토어 — B2C 자재 유통
│   ├── 타겟: 셀프시공 개인, 소규모 시공업체, 건물관리자
│   ├── 판매 채널
│   │   ├── 아임웹 (자사몰)
│   │   ├── 사방넷 (통합관리)
│   │   ├── 오늘의집 (마켓플레이스)
│   │   └── 쿠팡 (마켓플레이스)

    ├── 타겟: 일반 가정, 인테리어 관심 소비자
```

## §2. 영업 파이프라인

### 공통 파이프라인 (POUR솔루션, POUR스토어, 그로홈)
```
신규 → 상담중 → 견적제출 → 계약예정 → 계약완료
 │                                        │
 └── 보류/이탈 ←──────────────────────────┘
```

| 상태 | 설명 | 액션 |
|------|------|------|
| 신규 | 리드 유입 (박람회/사이트/전화) | 담당자 배정, 1차 연락 |
| 상담중 | 고객과 소통 진행 중 | 니즈 파악, 현장 확인 |
| 견적제출 | 견적서 전달 완료 | 후속 연락, 조건 협의 |
| 계약예정 | 계약 합의, 서류 진행 | 계약서 준비, 일정 조율 |
| 계약완료 | 계약 체결 | 시공 일정, 자재 발주 |
| 보류 | 일시 중단 | 재연락 일정 설정 |

### POUR공법 추가 파이프라인 (입찰)
```
입찰등록 → 투찰완료 → 낙찰 → 계약완료
                    └→ 유찰 → (재입찰 또는 종료)
```

| 상태 | 설명 |
|------|------|
| 입찰등록 | 나라장터/조달청 입찰 정보 등록 |
| 투찰완료 | 가격/기술 제안서 제출 |
| 낙찰 | 선정, 계약 협의 진행 |
| 유찰 | 미선정, 원인 분석 후 재입찰 검토 |

## §3. 자사몰 사이트 운영 구조

### 사이트 현황
| 사이트 | 플랫폼 | 브랜드 컬러 | URL |
|--------|--------|------------|-----|
| POUR솔루션 | 아임웹/카페24 | 네이비 + 블루 | (자사몰) |
| POUR공법 | 아임웹/카페24 | 네이비 + 오렌지 | (자사몰) |
| POUR스토어 | 아임웹/카페24 | 네이비 + 그린 | (자사몰) |

### Firestore 연동 페이지 구조
각 자사몰에 iframe 또는 직접 링크로 임베드하는 독립 페이지:

```
site-solution.html (POUR솔루션)
├── 파트너사 문의/입점신청 폼 → Firestore partner-inquiries
├── 대리점 문의 폼 → Firestore dealer-inquiries
├── 공신력 수치 표출 위젯 → Firestore site-metrics (읽기전용)
└── 소개 페이지 (회사/공법 안내)

site-method.html (POUR공법)
├── 시공사 문의/기술제안 요청 폼 → Firestore partner-inquiries (brand:'method')
├── 공신력 수치 표출 위젯 → Firestore site-metrics (읽기전용)
└── 공법 소개 페이지

site-store.html (POUR스토어)
├── 대리점/유통 문의 폼 → Firestore dealer-inquiries (brand:'store')
├── 공신력 수치 표출 위젯 → Firestore site-metrics (읽기전용)
├── 제품 문의 폼
└── 셀프시공 상담 요청
```

### 사이트 ↔ 관리센터 데이터 흐름
```
[사이트 페이지]                         [관리센터 admin.html]
파트너사 문의 폼 ──write──→ partner-inquiries ──read──→ 문의접수 관리
대리점 문의 폼  ──write──→ dealer-inquiries  ──read──→ 문의접수 관리
                                                         │
                                               승인 시 SMS 발송
                                               서류 제출 안내
                                               계약 관리
                                                         │
수치 표출 위젯  ──read──← site-metrics ────write──← 공신력 수치 관리
```

### 문의 처리 워크플로우
```
1차 문의 접수 (사이트 폼)
  → 관리센터 실시간 알림
  → 담당자 배정
  → 상태: 신규 → 검토중 → 승인 → 계약
  → 승인 시: SMS 자동 발송 (서류 제출 안내)
  → 서류 제출 확인
  → 계약 관리
```

### 공신력 수치 관리
관리센터에서 입력, 3개 사이트 일괄 실시간 반영:
- 누적 시공 세대수 (예: 2,600,000세대)
- 특허/인증 수 (예: 70여 개)
- 파트너사 수 (예: 250여 곳)
- 누적 시공 면적 (예: 1,500,000㎡)
- 제품 수 (예: 110여 개+)
- 협력사 수

## §5. (추후) AI 시공매칭

```
고객 시공 신청 → AI가 파트너사 DB에서 최적 매칭 추출
  → 고객에게 3~5개 파트너사 추천
  → 파트너사에 매칭 알림
  → 견적 비교 → 계약
```

매칭 기준: 지역, 공법 전문성, 시공 실적, 평점, 가용 일정
